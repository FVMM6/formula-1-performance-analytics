# Formula 1 Performance Analytics

Проект по анализу данных Formula 1, сфокусированный на результатах гонок, стартовых позициях, силе конструкторов, надёжности болидов и изменениях между историческими эпохами Формулы-1.

## Описание проекта

В этом проекте анализируются исторические данные Formula 1 с использованием Python и Jupyter Notebook. Основная цель проекта — понять, как стартовая позиция, конструктор, трасса и историческая эпоха связаны с итоговым результатом гонки.

Проект объединяет несколько сырых реляционных CSV-таблиц в один аналитический датасет и включает очистку данных, создание новых признаков, разведочный анализ данных, визуализации и проверку гипотез.

## Основные исследовательские вопросы

Проект отвечает на следующие вопросы:

1. Стала ли стартовая позиция сильнее связана с итоговой позицией в современной Formula 1?
2. Доминируют ли топ-конструкторы сразу по нескольким метрикам, а не только по общему количеству очков?
3. Увеличилась ли доля классифицированных финишей со временем, что может указывать на рост надёжности современной Formula 1?

## Датасет

В проекте используется датасет Formula 1 Race Data с Kaggle:

https://www.kaggle.com/datasets/jtrotman/formula-1-race-data?resource=download

Датасет организован как реляционная база данных, экспортированная в несколько CSV-файлов. В проекте используются следующие основные таблицы:

* `results.csv`
* `races.csv`
* `drivers.csv`
* `constructors.csv`
* `circuits.csv`
* `status.csv`

Исходный датасет содержит скрытые пропуски, закодированные как `\N`, разные типы данных в некоторых колонках и несколько таблиц, которые необходимо объединить перед анализом. Поэтому очистка и подготовка данных являются важной частью проекта.

Notebook автоматически определяет максимальный год, доступный в загруженных данных, и создаёт исторические эпохи на основе фактического диапазона лет.

## Подготовка данных

Основной аналитический датасет создаётся путём объединения нескольких исходных CSV-таблиц:

```text
results
+ races
+ drivers
+ constructors
+ circuits
+ status
```

Процесс подготовки данных включает:

* загрузку нескольких CSV-файлов;
* поиск скрытых пропусков, закодированных как `\N`;
* замену `\N` на корректные пропущенные значения;
* выбор релевантных колонок;
* переименование колонок, чтобы избежать конфликтов после объединения таблиц;
* преобразование числовых полей к правильным типам данных;
* объединение реляционных таблиц в один основной аналитический датасет;
* сохранение обработанного датасета локально.

## Созданные признаки

В проекте создаются новые признаки для анализа:

| Признак                | Описание                                               |
| ---------------------- | ------------------------------------------------------ |
| `driver_name`          | Полное имя пилота                                      |
| `position_gain`        | Разница между стартовой позицией и итоговой позицией   |
| `is_podium`            | Финишировал ли пилот в топ-3                           |
| `is_points_finish`     | Набрал ли пилот очки                                   |
| `is_classified_finish` | Был ли пилот классифицирован на финише                 |
| `era`                  | Историческая эпоха Formula 1                           |
| `grid_group`           | Группа стартовой позиции: Top 5, P6-P10, P11-P15, P16+ |

## Гипотезы

### Гипотеза 1

**Стартовая позиция сильнее связана с итоговой позицией в современной Formula 1, чем в более ранние эпохи.**

В рамках анализа сравнивается связь между стартовой позицией и итоговой позицией по историческим эпохам. Так как обе переменные по сути являются ранговыми, в качестве основной метрики используется корреляция Спирмена.

Результаты показывают, что связь между стартовой и итоговой позицией сильнее в современной Formula 1, особенно в самой поздней эпохе.

### Гипотеза 2

**Топ-конструкторы имеют более высокие средние очки и долю подиумов, но не обязательно имеют самый высокий средний прирост позиций.**

В анализе конструкторы сравниваются по нескольким метрикам: общие очки, средние очки, доля подиумов и средний прирост позиций.

Результаты показывают, что доминирующие конструкторы действительно сильны по очкам и подиумам, однако средний прирост позиций отражает другую сторону гоночной эффективности. Топ-команды часто стартуют ближе к первым позициям, поэтому у них меньше возможностей отыграть много мест по ходу гонки.

### Гипотеза 3

**Доля классифицированных финишей увеличилась со временем, что может указывать на рост надёжности современной Formula 1.**

В анализе рассчитывается доля классифицированных финишей по годам и по историческим эпохам. Классифицированным финишем считается результат со статусом `Finished` или статусом, содержащим `Lap`, например `+1 Lap` или `+2 Laps`.

Результаты показывают, что доля классифицированных финишей существенно выросла со временем, особенно начиная с 2000-х годов. Это согласуется с идеей о повышении надёжности современной Formula 1.

## Визуализации

В проекте используются следующие визуализации:

* распределение прироста позиций;
* стартовая позиция против итоговой позиции;
* среднее количество очков по годам;
* корреляционная матрица числовых переменных;
* прирост позиций по историческим эпохам;
* доля подиумов по группам стартовой позиции;
* топ-конструкторы по общему количеству очков;
* доля подиумов у топ-конструкторов;
* средний прирост позиций у топ-конструкторов;
* доля классифицированных финишей по эпохам;
* доля классифицированных финишей по годам.

## Основные выводы

* Стартовая позиция сильнее связана с итоговым результатом в современной Formula 1.
* Топ-конструкторы обычно имеют больше очков и более высокую долю подиумов, но не обязательно имеют высокий средний прирост позиций.
* Доля классифицированных финишей существенно выросла со временем, что согласуется с ростом надёжности современной Formula 1.
* Прирост позиций нужно интерпретировать осторожно, потому что сильные команды часто стартуют впереди и имеют меньше возможностей отыграть много мест.
* Сырые данные Formula 1 требуют очистки перед анализом, потому что пропуски могут быть закодированы как `\N`, а некоторые числовые колонки могут быть загружены как строки.

## Как запустить проект

1. Склонировать репозиторий:

```bash
git clone https://github.com/FVMM6/formula-1-performance-analytics.git
cd formula-1-performance-analytics
```

2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Скачать датасет с Kaggle и поместить все CSV-файлы в папку:

```text
data/raw/
```

4. Собрать обработанный датасет:

```bash
python src/build_dataset.py
```

5. Запустить разведочный анализ:

```bash
python src/explore_dataset.py
```

6. Запустить проверку гипотез:

```bash
python src/hypothesis_checks.py
```

7. Открыть финальный notebook-report:

```text
notebooks/formula_1_project.ipynb
```

## Примечания

Исходные файлы датасета не хранятся в этом репозитории, потому что они скачиваются с Kaggle и игнорируются Git. Чтобы воспроизвести анализ, необходимо вручную скачать датасет и поместить все CSV-файлы в папку `data/raw/`.

Обработанный датасет и графики можно сгенерировать локально, запустив скрипты из папки `src` или выполнив Jupyter Notebook.
